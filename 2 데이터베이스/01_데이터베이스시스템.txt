데이터 : 관찰의 결과로 나타난 정량적, 정성적인 실제값
정보 : 데이터의 의미를 부여한 것
지식 : 사물이나 현상에 대한 이해
데이터 베이스 : 조직에 필요한 정보를 얻기 위해서 논리적으로 연관된 데이터를 구조적으로 통합해 놓은 것
			-통합데이터, 저장데이터, 공유데이터 , 운영데이터

데이터베이스 특징
-실시간 접근성
-계속적인 변화
-동시 공유
- 내용에 따른 참조

*레코드 : 테이블의 최소 단위
4차 산업혁명 : 커스텀화

데이터베이스 언어
-SQL(Structured Query Lanuage) 구조화된 질의어

SELECT 필드1, 필드2.....  FROM 테이블명 WHERE ;
INSERT INTO 테이블명 VALUES( , , , );

-데이터 정의아(DDL) : CREATE, ALTER, DROP ( 테이블, 인덱스, 뷰, 시퀀스)
-데이터 조작어 (DML) : INSERT, DELETE, UPDATE
-데이터 질의어(DQL) : SELECT
-데이터 제어어(DCL) : COMMINT, ROLLBACK(TCL), GRANT, REVOKE


관계 데이터 모델
-1970년대 에드가 코드 박사 : 수학의 집합이론에 근거해서 만들어진 모델
관계데이터 모딜+ SQL : 원하는 데이터를 쉽게 표현이 가능하여 사용자가 원하는 정보를 제공

!!릴레이션(relation) (관계데이터모델에선 테이블이 아닌 릴레이션이라함)
행(row)과 열(column)로 이루어진 테이블(table) : 관계, 친척....

-relational data model : 관계 데이터 모델 
-relational database :  관계 데이터베이스
-relational algerbra : 관계 대수

*FK :외부 참조

기능명세 -> 데이터 정의 -> 데이터베이스 구조 설계


=====정보시스템 구축 절차=====
1)분석 : 시스템분석, 요구사항 분석 (우리가 무엇을(what?) 결정)
	   타당성 근거 제시 : 사용자의 인터뷰, 조사 필요 ===> 산출물 (문서)
2)설계 : 시스템 설계, 프로그램 설계 (우리가 결정한 시스템을 "어떻게 how" 결정
3)구현 : 산출물을 토대로 프로그래머(코더)에게 넘겨 주면 설계도 대로 만들어야 한다.
4)테스트 : 검수
5)유지보수
