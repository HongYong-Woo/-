릴레이션 스키마
-속성 : 릴레이션 스키마의 열
-도메인 :속성이 가실 수 있는 값의 집합
-차수 : 속성의 개수

-투플 : 릴레이션의 행
-카디날리티 : 투플의 수

릴레이션의 특징
1. 속성은 단일 값을 가진다.
2. 속성은 서로 다른 이름을 가진다.
3. 한 속성의 값은 모드 같은 도메인 값을 가진다.
4. 속성의 순서는 상관없다.
5. 릴레이션 내의 중복된 투플은 허용하지 않는다.
6. 투플의 순서는 상관 없다.

무결성 : 중복을 최소화
-기본 키(PK) : 투플을 구분하기 위해 사용 => 고유한 값(unique), null 허용X(not null)
-슈퍼 키(SK) : PK가 될 가능성이 있는 값
-후보 키(CK) : 투플을 유일하게 식별 할 수 있는 속성의 최소 집합(복합키)
-대리 키(SK) : 기본키가 보안을 요하거나 여러개의 속성으로 구성되어 복할 하거나 할때 가상의 속성을 만들어 기본키로 삼음 ( 인조키 )
-대체 키(AK) : 기본키로 선정되지 않은 후보키를 말함
-외래 키(FK) : 다른 릴레이션의 기본키를 속성을 말함( 참조 )

*데이터 무결성 : 일관성과 정확성을 지키는것
*무결성 제약 조건
1. 도메인 무결성 제약조건 : 도메인 제약 => 지정된 값만 가져야함
2. 개체 무결성 제약조건 : 기본키 제약 => 고유한 값, not null
3. 참조 무결성 제약조건 : 외래키 제약 => 부모 릴레이션의 제약을 받음
